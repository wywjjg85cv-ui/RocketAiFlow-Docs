# Pre-Provisioned Dashboards

RocketAiFlow includes pre-provisioned Grafana dashboards as part of its observability and
monitoring setup.

This is a practical operational advantage: teams can start from a ready dashboard foundation
instead of building the core monitoring layer from scratch before they can diagnose live behavior.

## What pre-provisioned dashboards mean

In practice, pre-provisioned dashboards mean that the first operational views are already prepared
for the deployment. Teams do not need to design the initial Grafana structure just to get baseline
visibility into infrastructure, telephony, containers, and database state.

## Why they matter

They help teams:

- gain operational visibility faster after deployment
- start troubleshooting from a known dashboard baseline
- reduce the time spent assembling first-pass monitoring panels
- align operations, infrastructure, and telephony teams around shared views

## Included dashboard categories

RocketAiFlow can come with pre-provisioned dashboard categories such as:

- `Node Exporter Full`
- `Asterisk Log`
- `Asterisk Overview`
- `cAdvisor`
- `MySQL 8`

These views help teams get faster visibility into:

- host and system health
- telephony activity
- Asterisk behavior and logs
- container runtime monitoring
- database visibility

## When to use each one

- Use `Node Exporter Full` when a workflow issue may come from the host or system layer.
- Use `Asterisk Log` when you need detailed telephony-related log review.
- Use `Asterisk Overview` when you need a higher-level picture of Asterisk behavior.
- Use `cAdvisor` when the deployment runs in containers and runtime pressure may be involved.
- Use `MySQL 8` when persistence, data access, or backend response behavior needs investigation.

## How they fit into daily operations

In daily operations, these dashboards give teams a stronger monitoring starting point out of the
box. They are useful for both routine health checks and incident investigation.

A practical operating pattern is:

1. review high-level live signals in Monitoring and Visibility
2. move into the most relevant pre-provisioned dashboard category
3. confirm whether the issue is infrastructure, telephony, runtime, or database related
4. use logs, traces, and call records to narrow the investigation further
