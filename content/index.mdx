## Why teams use these docs

RocketAiFlow documentation is not meant to read like a static feature list. It is designed to help
technical leads, solutions engineers, operators, and buyers understand whether the platform is
clear to implement and credible to run.

The docs emphasize:

- fast onboarding without skipping operational detail
- platform-agnostic architecture and provider flexibility
- flexible telephony architecture for real SIP and PBX environments
- practical setup paths for inbound and outbound workflows
- API extensibility and custom business actions
- Dedicated and Self-Hosted deployment flexibility with telephony fit
- observability, logs, traces, and troubleshooting readiness

## Evaluation checklist

Use the docs to confirm these questions early:

- Can one agent support a real inbound or outbound workflow without rigid setup?
- Can functions and API actions map to actual business steps?
- Can workflow context and dynamic parameters be passed cleanly?
- Can the platform fit the Dedicated or Self-Hosted model and the telephony environment you operate?
- Can operators monitor campaign state, endpoint health, logs, and traces without guessing?
