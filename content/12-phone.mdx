# Phone

The `Phone` area is useful when you want to test call behavior manually from inside the platform.

This is often the fastest way to validate prompt, voice, transfer behavior, and basic flow logic
before relying on a live inbound route or a larger outbound campaign.

## When To Use Phone

Use the Phone area when you want to:

- run a focused manual test
- validate the first prompt revision
- check whether transfer logic behaves correctly
- compare behavior before and after a config change

## What To Validate During A Test Call

During the call, confirm:

- the agent answers correctly
- the voice is working
- the prompt stays on task
- the transfer or hangup behavior matches the intended flow

## A Good Manual Test Pattern

For a first implementation:

1. make one change
2. place one controlled test call
3. review the result in Call Records
4. repeat only if the outcome is clear

This prevents confusion when multiple settings change at once.
