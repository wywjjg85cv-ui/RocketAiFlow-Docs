# Import Contacts

Import Contacts is the fastest way to prepare an outbound campaign with a controlled test list.

The first goal is accuracy, not volume. A clean import makes campaign testing much easier.

## Prepare The CSV First

Use a simple CSV with fields such as:

- `phone`
- `name`
- `surname`
- `data`

For the first pass, keep the file small and easy to validate manually.

## Minimum Mapping

Map at least the required field:

- `Phone`

If you include optional fields, make sure the column names are clear and the data is consistent
before upload.

## Track Import Progress

After the import starts, review the status using:

- `Processed`
- `Inserted`
- `Failed`
- `Skipped`

These numbers tell you whether the file was accepted cleanly or whether specific rows need review.

## Common Import Issues

- phone values are missing or malformed
- the wrong column is mapped to `Phone`
- the CSV contains duplicate or incomplete rows
- the campaign is started before the import is fully reviewed

## Validation Checklist

Before using the contacts in a campaign, confirm:

- the expected number of contacts was processed
- valid rows were inserted
- failed rows are understood
- the mapped phone data is usable for outbound calling
