# Configure Agent Prompt

The prompt defines how the agent should behave during the call. A clear prompt is usually more
useful than a long prompt.

For the first version, focus on role, tone, allowed actions, and escalation behavior.

## What A Good First Prompt Should Cover

Your prompt should tell the model:

- who it is
- why it is on the call
- how it should speak
- what information it should collect
- when it should transfer or end the conversation

## Starter Prompt Example

```text
You are Rocket Demo Assistant, the AI voice assistant for RocketAiFlow.
Speak clearly, briefly, and professionally.
Help users understand our product, answer simple questions, and collect contact details when useful.
If the caller asks to speak with a human sales representative, transfer the call.
```

This is enough for a first controlled test.

## Keep The Prompt Operational

A practical prompt usually works better when it includes:

- a short role definition
- one or two conversational style rules
- a clear business objective
- a clear human handoff condition

Avoid turning the first prompt into a policy document. Add detail only when a real test shows a
gap.

## What To Improve After The First Calls

Once you have test data, refine the prompt to improve:

- response brevity
- data collection order
- transfer timing
- how the agent handles unsupported questions

## Prompt and functions should reinforce each other

In RocketAiFlow, the prompt is not isolated from the function layer. The prompt should make it
clear:

- what actions the agent is allowed to trigger
- when external lookups or API actions should be used
- which details should be collected before a function call
- when runtime context should shape the next step

For example, an appointment flow may collect the caller name, preferred date, and service type
before calling a booking function. A lead qualification flow may collect company size or interest
level before sending a CRM update.

## Prompt Review Checklist

Before moving on, confirm:

- the role is clear
- the language is aligned with the intended caller
- the business goal is explicit
- the transfer condition is stated
