# Logs Overview

Logs help teams move from symptoms to events.

Dashboards usually tell you that something changed. Logs help explain what changed, when it
happened, and which system layer produced the signal.

## Where logs fit

In RocketAiFlow operations, logs are most useful when combined with:

- call records
- pre-provisioned Grafana dashboards
- endpoint and trunk monitoring
- traces where available

## Asterisk log visibility

The `Asterisk Log` dashboard is a particularly useful starting point for telephony-related log
activity. It helps teams correlate operational log signals with call-path issues and Asterisk
behavior.

## When to review logs

Review logs when:

- a dashboard shows a shift but not the event behind it
- telephony behavior needs event-level confirmation
- troubleshooting requires exact incident timing
