# LLM / Model Providers

The model layer helps interpret inputs, shape responses, and support controlled reasoning within
the workflow. It should be selected based on the actual task, not only on general model reputation.

## Model-layer considerations

- how much reasoning is actually needed for the workflow
- latency tolerance in live calls
- prompt and control requirements
- deployment or customer constraints around where inference runs

## Design principle

Use the model layer as part of a controlled workflow system. The model should support the business
path, not replace the workflow boundary that keeps the interaction operationally reliable.
