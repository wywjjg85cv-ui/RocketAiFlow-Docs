# AI Dialer Flows

AI Dialer Flows is where outbound calling campaigns are created and managed.

For a first implementation, the goal is not scale. The goal is to confirm that one campaign, one
contact set, and one agent work together correctly.

## Before You Create A Campaign

Confirm these inputs first:

- the agent is already configured
- the prompt and functions support the outbound use case
- contacts are available or ready for import
- the trunk configuration supports outbound calling

## Example First Campaign

Use values like:

- `Campaign name`: `Q2 Demo Outreach`
- `Timezone`: `Europe/Rome`
- `Purpose`: call leads who requested a product demo

This is a realistic first scenario because the call objective is narrow and easy to verify.

## Keep The First Campaign Controlled

A safer first rollout usually means:

- a small test list
- a clear call purpose
- one primary agent
- a limited test window
- conservative calls-per-second settings
- a conservative maximum active-calls limit per campaign

This makes it easier to identify whether an issue comes from campaign setup, contacts, prompt
behavior, or telephony configuration.

## Pacing and live execution controls

Before scaling a campaign, set:

- a `calls per second` value that controls launch pressure
- a `maximum active calls per campaign` value that limits live concurrency

These controls matter because they affect more than raw throughput. They also shape:

- telephony pressure
- how quickly active calls build
- how stable campaign outcomes look during launch
- how easy it is to diagnose problems when behavior drifts

Start conservatively and increase only after the first calls and metrics look healthy.

## What To Verify Before Launch

Before starting the first outbound test, confirm:

- the campaign uses the correct agent
- the timezone is correct
- the contact source is ready
- the outbound path is supported by the configured trunk
- the initial pacing and active-call limits match the environment you are testing

## After The First Calls

Review call records to confirm:

- calls actually started
- the agent handled the call objective
- outcomes match the campaign purpose
- transfer or hangup behavior was correct
- talk time and ring time look reasonable for the workflow
- transcripts or recordings are reviewable when they are enabled

When the outbound workflow uses custom actions, also confirm that campaign-specific values, contact
fields, and runtime data are being passed correctly into those functions. This matters for use cases
such as CRM updates, follow-up tasks, and external lead qualification checks.
