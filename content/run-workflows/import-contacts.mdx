# Import Contacts

Import Contacts lets you upload a CSV and decide where those contacts should be used.

You can associate imported contacts with:

- a campaign, when the contacts must be called by an outbound dialer flow
- an agent, when inbound calls should recover contact data through `Contact Inbound`

The first goal is accuracy, not volume. A clean import makes campaign testing much easier.

## Prepare The CSV First

Use a simple CSV with fields such as:

- `phone`
- `contactId`
- `name`
- `surname`
- `data`
- `priority`
- `scheduledAt`

For the first pass, keep the file small and easy to validate manually.

## Select The Correct Usage

When importing contacts, choose the usage based on where the list should be available:

- use `AI Dialer Flows` when the contacts must be attached to an outbound campaign
- use the agent/inbound contact usage when the contacts must be available to an agent for inbound lookup

Do not select a campaign usage for contacts that are only needed for inbound caller lookup, and do
not select an inbound agent usage for contacts that must be dialed by an outbound campaign.

## Associate Contacts With A Campaign Or Agent [#associate-contacts-with-a-campaign-or-agent]

Use campaign association when you want the dialer to call the contacts. In that case, the contact
belongs to the campaign list and can provide values for greeting, prompt variables, function
context, and call review.

Use agent association when you want inbound calls to recover contact information. In that case, the
contact list is loaded for the agent through `Contact Inbound`.

For inbound rendering, both conditions must be true:

1. the agent settings allow contact lookup from `Contact Inbound`
2. `Contact Inbound` contains a loaded list with a contact matching the caller phone number

When those conditions are true, the contact template used by the agent can render variables from the
matched contact. If a variable does not exist on that contact, it is not rendered.

For outbound contacts, make sure the CSV includes the values you want to render dynamically. Custom
values should be stored as key-value pairs inside the `data` object, so template variables can use
paths such as `{t.data.cf}`.

## Minimum Mapping

Map at least the required field:

- `Phone`

All other fields are optional. If you include optional fields, make sure the column names are
clear and the data is consistent before upload.

## Contact Fields

| Field | Required | Purpose |
| --- | --- | --- |
| `phone` | Yes | Destination phone number used for the outbound call. |
| `contactId` | No | External identifier for the contact. Use it to correlate RocketAiFlow contacts and calls with records in your CRM, database, or source system. |
| `name` | No | Contact first name, available for personalization and context. |
| `surname` | No | Contact surname, available for personalization and context. |
| `data` | No | Free JSON object for custom fields. Values placed here are available during the call and can be used as agent context. |
| `priority` | No | Contact priority. If omitted, RocketAiFlow loads the contact with priority `1`. Contacts with a higher priority are selected first when the campaign chooses the next eligible contact. |
| `scheduledAt` | No | Date and time when the contact becomes eligible for calling. If `scheduledAt` is greater than the current time while the campaign is in progress, the call waits until that scheduled time before it can start. |

## Track Import Progress

After the import starts, review the status using:

- `Processed`
- `Inserted`
- `Failed`
- `Skipped`

These numbers tell you whether the file was accepted cleanly or whether specific rows need review.

## Common Import Issues

- phone values are missing or malformed
- the wrong column is mapped to `Phone`
- the CSV contains duplicate or incomplete rows
- the campaign is started before the import is fully reviewed

## Validation Checklist

Before using the contacts in a campaign, confirm:

- the expected number of contacts was processed
- valid rows were inserted
- failed rows are understood
- the mapped phone data is usable for outbound calling
