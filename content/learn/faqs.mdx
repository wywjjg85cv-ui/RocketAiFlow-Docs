# FAQs

### Is RocketAiFlow only for technical teams?

No. The platform needs technical ownership to be implemented well, but the workflows themselves are
operational systems and should also make sense to operations, CX, revenue, or service leaders.

### Does RocketAiFlow require a single provider stack?

No. The documentation positions RocketAiFlow as platform-agnostic by design. Implementation choices
should be driven by fit, constraints, and customer requirements.

### Can one workflow cover every call type?

Usually no. Better results come from smaller, bounded workflows with explicit handoff paths than
from one oversized flow that tries to handle everything.

### Should automation always maximize containment?

Not always. Some workflows should optimize for rapid qualification or fast transfer rather than full
containment. The right metric depends on the business objective.

### Where should detailed implementation docs live?

Detailed setup should live in deployment, integration, and future API reference sections, not in
high-level product overview pages.
