# Getting Started

RocketAiFlow helps you build AI-powered voice workflows for inbound and outbound calls.

In this guide, you will create a real working setup:

- one AI voice agent
- one inbound routing flow
- one outbound campaign
- one contact import
- one test call
- one call record review

By the end of this setup, you will understand how the main areas of the product work together.

## What You Will Build

You will create a sample voice assistant called `Rocket Demo Assistant`.

This assistant will:

- answer inbound calls
- speak with leads in a natural voice
- collect basic information
- transfer the call to a human team when needed
- support outbound calling campaigns

## Core Product Areas

RocketAiFlow is organized into a few main sections:

- `AI Voice Agents`
  Create and configure the voice agent, prompt, model, voice, and functions.

- `AI Inbound Routing`
  Decide how inbound calls are routed to an AI agent.

- `AI Dialer Flows`
  Manage outbound campaigns, contacts, and call execution.

- `Import Contacts`
  Upload CSV files and map contact fields for campaigns.

- `Call Records`
  Review call outcomes, timestamps, and call-level details.

- `Trunk`
  Configure SIP trunk connectivity for inbound and outbound calling.

- `Phone`
  Test calls manually from inside the platform.

## Before You Start

Make sure you have:

- access to the RocketAiFlow platform
- at least one configured SIP trunk
- permissions to create agents and campaigns
- a CSV file with test contacts if you want to try outbound flows

If you only want to test inbound first, you can skip contact import and campaign setup.

## Quick Start Path

Follow this order for the smoothest setup:

1. Create an AI voice agent
2. Configure the prompt
3. Add the required functions
4. Connect the agent to inbound routing
5. Create an outbound campaign
6. Import contacts
7. Place a test call
8. Review the result in Call Records

## Step 1: Create Your First AI Voice Agent

Open `AI Voice Agents` and create a new agent.

Use these sample values:

- `Name`: `Rocket Demo Assistant`
- `Language`: `English`
- `Goal`: answer calls, qualify the caller, and transfer to sales when appropriate

This agent will be the base of both your inbound and outbound flows.

## Step 2: Add a Prompt

Use a simple first prompt like this:

```text
You are Rocket Demo Assistant, the AI voice assistant for RocketAiFlow.
Speak clearly, briefly, and professionally.
Help users understand our product, answer simple questions, and collect contact details when useful.
If the caller asks to speak with a human sales representative, transfer the call.
```

This gives the model a clear operational role from the beginning.

<ImagePlaceholder
  title="Prompt configuration area"

## Step 3: Add Functions

Start with these functions:

- `hangup_call`
  End the call cleanly when the conversation is complete.

- `transfer_call`
  Transfer the caller to a human extension.

- optional: `get_weather`
  A sample custom API function useful for testing function calling behavior.

For transfer flows, use `exten` as the main parameter.

<ImagePlaceholder
  title="Functions configuration screen"
  description="Functions configuration screen with hangup, transfer, and an optional sample API function."
/>

## Step 4: Connect Inbound Routing

Open `AI Inbound Routing` and create or edit a route.

Assign the inbound route to `Rocket Demo Assistant`.

Now, when an inbound call reaches that route, the AI agent will answer it.

<ImagePlaceholder
  title="Inbound routing configuration"
  description="Inbound routing configuration screen with Rocket Demo Assistant assigned to a route."
/>

## Step 5: Create an Outbound Campaign

Open `AI Dialer Flows` and create a campaign.

Example:

- `Campaign name`: `Q2 Demo Outreach`
- `Timezone`: `Europe/Rome`
- `Purpose`: call leads who requested a product demo

This campaign will be used for outbound call automation.

<ImagePlaceholder
  title="Outbound campaign creation"
  description="Outbound campaign creation screen for Q2 Demo Outreach."
/>

## Step 6: Import Contacts

Open `Import Contacts` and upload a CSV file.

Example columns:

- `phone`
- `name`
- `surname`
- `data`

Map at least the required field:

- `Phone`

After the import starts, you can track progress through:

- `Processed`
- `Inserted`
- `Failed`
- `Skipped`

<ImagePlaceholder
  title="CSV import and field mapping"
  description="CSV import and field mapping screen for phone, name, surname, and data columns."
/>

## Step 7: Test the Flow

You can test the setup in two ways:

- place a real inbound call to the configured route
- use the internal `Phone` page for manual testing

This helps confirm:

- the agent answers correctly
- the voice is working
- the prompt behaves as expected
- transfer logic works

<ImagePlaceholder
  title="Phone test page"
  description="Phone test page or live test area used to validate the first call flow."

## Step 8: Review Call Records

Open `Call Records` after the test.

Use this page to verify:

- call status
- timestamps
- outcome
- transcript or recording data if available

This is the main place to validate whether your workflow is behaving correctly.

Your first setup is complete when:

- one agent exists
- one prompt is configured
- one transfer or hangup function is available

- one campaign exists
- one contact import has been completed
- one call appears in `Call Records`
